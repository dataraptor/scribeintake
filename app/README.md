# app

The client application — the user-facing frontend.

A single cross-platform codebase that builds to multiple targets (mobile, web, and
more). It renders what the backend returns and handles user interaction; it holds no
business logic of its own. The shared source lives in `lib/`; the per-platform build
folders are generated by the framework.

**Contains:** the frontend source, UI components, app configuration, and tests.

**Depends on:** `api` (over the network, at runtime).
